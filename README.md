# AI Text Humanizer — RL-Trained Detector Evasion

A self-hosted AI text humanizer that **actually works** against modern detectors (Copyleaks, Originality.ai, GPTZero, Turnitin). Based on the AuthorMist and StealthRL research papers.

## Why This Works (When Prompting Doesn't)

Asking an LLM to "rewrite this to sound human" fails because the output is still generated by an LLM with the same token probability distributions and statistical fingerprints. Detectors catch the distribution, not the words.

This project uses **reinforcement learning** to fine-tune a small language model (Qwen2.5-3B) where the reward signal comes directly from detector APIs. The model learns to produce text that detectors classify as human by optimizing against the detectors themselves.

**Research results:**
- AuthorMist: 78-96% attack success rate, >0.94 semantic similarity
- StealthRL: 99.9% attack success rate at 1% FPR
- Both outperform DIPPER (11B params) with only 3-4B params

## Quick Start

```bash
pip install -r requirements.txt

# Option A: Use pre-trained AuthorMist (quick test)
python inference.py --model authormist/authormist-originality --input "your text"

# Option B: Train your own (better, current results)
python data/prepare_dataset.py --num-samples 5000
python train.py --detector gptzero --epochs 1

# Option C: Start web server
python server.py
```

## Hardware

- **Training**: 1x A100 40GB (~$2/hr on RunPod) or 2x RTX 4090
- **Inference**: Any 8GB+ VRAM GPU, or CPU with quantized model
